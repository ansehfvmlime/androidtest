# Task 002 — Summary UI Upgrade

## Context

현재 캐릭터 검색 결과 페이지의 summary 영역은
핵심 정보가 분산되어 있어 캐릭터 상태를 한눈에 파악하기 어렵다.

로펙의 grid 기반 UI 구조를 참고하여
캐릭터 핵심 정보를 보기 편한 layout으로 재구성한다.

이번 작업의 목적은 기능 추가가 아니라
UI / 정보 구조 개선이다.

---

# Page Structure

페이지는 다음 구조를 따른다.

1. 검색 영역
2. summary 영역
3. 네비게이션 탭
4. 메인 컨텐츠 grid

summary는 nav 위에 위치한다.

---

# Summary Layout

summary 영역에는 캐릭터 기본 정보를 표시한다.

표시 정보

- 캐릭터 이름
- 서버
- 직업
- 아이템 레벨
- 길드
- 칭호
- 캐릭터 프로필 이미지

UI 구조
┌─────────────────────────────┐
│ 캐릭터 이미지 │ 캐릭터 정보 │
│ │ 이름 │
│ │ 서버 │
│ │ 아이템 레벨 │
│ │ 직업 │
│ │ 길드 │
│ │ 칭호 │
└─────────────────────────────┘

---

# Main Grid Layout

summary 아래 메인 영역은 **2 column grid** 구조를 사용한다.

### Row 1

- 좌측: 전투력 카드
- 우측: 보석 카드

### Row 2

- 좌측: 특성 카드 (치명 / 특화 / 신속)
- 우측: 보석 카드 영역의 연장 또는 동일 column 유지

### Row 3

- 무기 / 방어구 카드
- 악세서리 / 스톤 카드

### Row 4

- 각인 카드 (full width)

grid 구조 예시

┌─────────────┬─────────────┐
│ 전투력 │ 보석 │
│ 특성 │ │
├─────────────┼─────────────┤
│ 무기/방어구 │ 악세서리/스톤 │
├───────────────────────────┤
│ 각인 │
└───────────────────────────┘

각 영역은 독립적인 **card UI** 형태로 표시한다.

---

# Card Roles

### 전투력 카드

표시 정보

- 전투력 (API 값)

주의

- 실제 게임 전투력과 차이가 있을 수 있음
- 이번 작업에서는 UI 표시만 수행

---

### 특성 카드

표시 정보

- 치명
- 특화
- 신속

특성 카드는 **전투력 카드 아래에 위치한다.**

---

### 보석 카드

표시 정보

- 보석 리스트
- 보석 레벨

보석 카드는 **우측 상단에 위치한다.**

---

### 무기 / 방어구 카드

표시 정보

- 무기
- 방어구

---

### 악세서리 / 스톤 카드

표시 정보

- 목걸이
- 귀걸이
- 반지
- 어빌리티 스톤

---

### 각인 카드

표시 정보

- 각인 목록
- 각인 레벨

각인 카드는 **full width 카드**로 표시한다.

---

# Null-safe Rules

다음 값은 존재하지 않을 수 있다.

- 길드
- 칭호
- 프로필 이미지
- 전투력

값이 없는 경우

- "-" 또는 placeholder 표시
- layout이 깨지지 않아야 한다

---

# Non Goal

이번 작업에서는 다음을 수행하지 않는다.

- 전투력 계산식 구현
- 계산 기능 구현
- 원정대 기능 구현
- API 구조 변경
- 서버 라우트 추가
- 장비 데이터 로직 변경
- 각인 데이터 로직 변경

---

# Done Criteria

다음 조건을 만족하면 task 완료로 간주한다.

- summary UI가 grid 스타일로 재구성된다
- 캐릭터 기본 정보가 명확하게 표시된다
- 전투력 카드 아래에 특성 카드가 배치된다
- 보석 카드가 우측 상단에 표시된다
- 장비 정보가 무기/방어구 | 악세서리/스톤으로 분리된다
- 각인 카드가 full width로 표시된다
- 길드 / 칭호 / 이미지 누락 시 layout이 깨지지 않는다
- 기존 페이지 렌더링이 깨지지 않는다

---

# Constraints

- 전체 파일 재작성 금지
- 기존 한글 텍스트 변경 금지
- 관련 없는 partial 수정 금지
- minimal diff 우선
