# Task 007 — Crit Rate Input System

## Context

치명타 계산기를 구현하기 위해
사용자가 직접 조정할 수 있는 치명타 관련 입력 시스템을 구현한다.

이번 작업에서는
API 기반 입력과 별도로
사용자가 직접 설정 가능한 치명타 적중률 관련 옵션을 제공한다.

이 입력값들은 이후 치명타 계산 엔진에서 사용된다.

---

# Goal

사용자가 다음 항목을 설정할 수 있도록 UI 입력 구조를 구현한다.

- 치명 스탯
- 아크패시브 치명타 적중률 옵션
- 악세서리 치명타 적중률
- 팔찌 치명타 적중률
- 아드레날린 치명타 적중률
- 직업 각인 치명타 적중률

---

# Global Rules

## 치명타 환산 계수

치명타 환산 계수는 사용자가 직접 수정할 수 없도록 한다.

계산식은 내부 로직에서만 사용한다.

---

## 치명타 상한

실제 게임에서는 치명타 적중률 상한이 100%이지만

추후 **뭉툭한 가시 계산기**를 구현할 때
치명타 적중률이 120%까지 올라가는 상황을 고려해야 하므로

치명타 계산기에서는 상한을 적용하지 않는다.

---

# Crit Stat Input

치명 스탯은 직접 숫자를 입력하지 않고

**아크그리드 레벨 조정 방식**으로 설정한다.

### 아크그리드 설정

- Lv 0 ~ Lv 30
- 레벨 1 증가 시 치명 스탯 +50

예시

```
Lv 0  → 0
Lv 1  → 50
Lv 10 → 500
Lv 30 → 1500
```

UI는 toggle 또는 slider 방식으로 구현한다.

---

# Ark Passive Crit Rate

다음 아크패시브 옵션을 선택할 수 있도록 한다.

각 옵션은 select input으로 구현한다.

## 예리한 감각

선택 값

- 0
- 1
- 2

## 혼신의 강타

선택 값

- 0
- 1
- 2

## 일격

선택 값

- 0
- 1
- 2

각 값은 이후 계산 로직에서 치명타 적중률 증가량으로 변환된다.

---

# Accessory Crit Rate

악세서리 치명타 적중률을 설정할 수 있도록 한다.

악세서리는 **2부위** 존재하므로

UI는 **1행 2열 구조**로 구현한다.

각 부위는 다음 옵션을 가진다.

| 옵션 | 치명타 적중률 |
| ---- | ------------- |
| 없음 | 0%            |
| 하   | 0.4%          |
| 중   | 0.95%         |
| 상   | 1.55%         |

### 색상 규칙

| 옵션 | 색상   |
| ---- | ------ |
| 없음 | 회색   |
| 하   | 파란색 |
| 중   | 보라색 |
| 상   | 노란색 |

---

# Bracelet Crit Rate

팔찌 치명타 옵션을 select input으로 구현한다.

총 **10개 옵션**을 제공한다.

## Option List

### 치명타 적중률 증가

- 없음
- 치명타 적중률 +3.4% (하)
- 치명타 적중률 +4.2% (중)
- 치명타 적중률 +5.0% (상)

### 치명타 피해 옵션

- 치명타 적중률 +3.4% / 치피 증가 (하)
- 치명타 적중률 +4.2% / 치피 증가 (중)
- 치명타 적중률 +5.0% / 치피 증가 (상)

### 치명타 저항 감소

- 치명타 저항 -1.8% (하)
- 치명타 저항 -2.1% (중)
- 치명타 저항 -2.5% (상)

---

## Special Rule

치명타 저항 감소 옵션은

치명타 적중률 증가와 동일한 기대값으로 계산한다.

예시

```
치명타 저항 -1.8% → 치명타 적중률 +3.4%
```

단

```
치명타 적중률 +3.4% ≠ 치명타 저항 -1.8%
```

즉 계산기에서는 동일한 값으로 사용하지만
실제 효과는 동일하지 않다.

# Class Engraving Crit Rate

직업 각인 치명타 적중률 입력을 제공한다.

현재는 **API 기반 자동 계산을 하지 않는다.**

사용자가 직접 입력한다.

예시

```
<input type="number">
```

향후

- 아크패시브
- 직업 각인
- 클래스 효과

API 데이터를 기반으로 자동 계산하도록 확장한다.
